iPhoneのScaniverseで撮影した部屋の3Dモデルを張り合わせて、大きな家全体の3Dモデルを作成して閲覧する方法をまとめる。

1. Scaniverse(OBJ形式出力)
2. CloudCompare(手動整列)
3. Blender(編集)
4. GLB形式出力
5. ブラウザ閲覧

## 前提・ゴール

* iPhoneのLiDARスキャン(Scaniverse)で、部屋ごとに3Dモデルを取得済み
* 目的:

  * **複数の部屋OBJを手動で位置合わせ**して家全体の形にする
  * Blenderで重複部分や不要部分を削除して綺麗にする
  * 最終的に **GLB(glTF)** を出力して、ブラウザ上で閲覧できるようにする

---

## 1. ScaniverseでOBJとして出力(iPhone内に保存)

1. Scaniverseで部屋のスキャンを開く
2. Export(共有/出力)から **OBJ** を選択
3. 保存先は iPhoneの **ファイル app**(ローカルやiCloud Driveなど)に保存
4. ScaniverseのOBJ出力は、通常 **ZIP(圧縮ファイル)** として保存される

> メモ: ScaniverseのOBJは「OBJ本体 + MTL + テクスチャ画像」等がセットになっていることが多く、ZIP化されるのが自然。

---

## 2. iPhone → Windows11へ転送(Dropbox等)

1. Dropbox / Google Drive / OneDrive などを介して、iPhoneのZIPをWindowsへ転送
2. Windows側でZIPを確認(サイズは数十MB程度が目安)
3. **各OBJごとにZIPを解凍**

   * 解凍すると、部屋ごとにフォルダが生成される(例: `Room_01/`)
   * フォルダ内に以下が入っていることが多い:

     * `*.obj`
     * `*.mtl`
     * `textures/` や `*.jpg` / `*.png` などのテクスチャ

> 注意: OBJは「OBJファイル単体」ではなく、**MTLとテクスチャの相対パス関係**が重要。
> フォルダ構造を崩すと、後で色/テクスチャが出ない原因になる。

---

## 3. CloudCompareでOBJを開く

1. CloudCompareを起動
2. `File > Open...` で、部屋フォルダ内の `*.obj` を開く
3. これを部屋数分繰り返し、複数OBJを同時に読み込む

> ここから先は、CloudCompareを「位置合わせ(手動)専門ツール」として使う。

---

## 4. CloudCompareで各OBJを手動で位置合わせ(ICP不要)

### 4.1 作業の基本方針

* 1つの部屋を「基準(動かさない)」に決める(例: Room_A)
* 他の部屋をそれに合わせて **手動で回転・移動**して繋ぐ
* 位置合わせ中は、見やすさのため **不要なOBJを不可視化**するのが効く

---

### 4.2 不要なOBJを不可視化する方法(CloudCompare)

CloudCompareの左側にある **DB Tree(オブジェクトツリー)** を使う。

* DB Tree内の各オブジェクトに対して、

  * **表示/非表示(目アイコンやチェック)** を切り替えられる(UIはバージョンで少し異なる)
* 手順(一般的な流れ):

  1. DB Treeで対象OBJを見つける
  2. 表示トグル(目アイコン / チェック)をOFFにして非表示にする
  3. 位置合わせ対象と基準部屋だけ表示して作業する

> コツ:
>
> * 「今動かしているOBJ」と「基準OBJ」だけ表示すると、重なりやズレが見やすい。
> * 他のOBJは一時的に全部OFFにしてよい。

---

### 4.3 手動で位置を調整する方法(CloudCompare)

CloudCompareでは、選択したオブジェクトに対して **回転・並進(移動)** を手動で行う。

基本の流れ:

1. DB Treeで「動かしたいOBJ」をクリックして選択
2. `Edit > Apply Transformation`(またはツールバーの変換/操作系)を使う

   * ここで **Translation(X/Y/Z移動)** と **Rotation(回転)** を指定して調整できる
3. 画面を見ながら、少しずつ値を変えて合わせる

   * ドア枠、壁の角、床の境界など「特徴が明確な場所」を基準に合わせる

> 重要:
>
> * ICPは使わない前提なので、「だいたい合うまで」この手動調整を繰り返す。
> * 「回転→移動→回転→移動」のように順番に追い込むと早い。

---

### 4.4 精度を上げるコツ(手動)

* **同じスケール感**になっているか(Scaniverse出力が部屋によって拡大縮小していないか)
* 合わせる基準点を複数見る:

  * 1箇所だけ合わせると、遠い場所でズレる
  * 角(コーナー)やドア枠など、離れた2点以上を見ると精度が上がる
* 画面がごちゃつくときは:

  * 位置合わせ対象以外を不可視化
  * こまめに視点を変える(斜めから・真上から・真横から)

---

## 5. 位置合わせ後の各OBJを、別ディレクトリへOBJで保存

位置合わせ後は、CloudCompareで「変換が適用された状態」を保存する。

1. DB Treeで保存したいOBJを選択
2. `File > Save` または `File > Save As...`
3. 形式は **OBJ(*.obj)** を選ぶ
4. 保存先は「整列済みOBJ用フォルダ」(例: `aligned_obj/`)を用意してそこへ保存

> 注意:
>
> * ここでも MTL/テクスチャ参照が壊れないように、保存後のフォルダ構造を確認する。
> * CloudCompare側の保存設定により、MTLやテクスチャの扱いが変わることがあるので、保存直後にBlenderで読み直して色が出るかチェックすると安全。

---

## 6. 保存した各OBJをBlenderで開く(位置が再現される)

1. Blender起動
2. `File > Import > Wavefront (.obj)`
3. `aligned_obj/` 内の**全部屋のobjファイルをインポート**
4. **CloudCompareで整列した位置が、そのままBlender上でも再現される**

> コツ:
>
> * 1回で複数OBJを選択して読み込んでもよい(環境によっては個別の方が安全)。
> * 読み込み直後は重いことがあるので、最初は1部屋ずつ表示して確認する。

---

## 7. Blenderで各OBJから不要部分を削除

### 7.1 Outliner(オブジェクト一覧)の出し方

「他の部屋を選びたい」「部屋ごとに表示/非表示したい」場合は Outliner を使う。

* 画面右上に Outliner が無い場合:

  1. 任意のエリア左上にある「エディタタイプ」アイコンをクリック
  2. `Outliner` を選択
* もしOutlinerを置く場所がない場合:

  1. 画面の端/角の分割操作でエリアを分割
  2. 分割した片方を Outliner にする

Outlinerでできること:

* 部屋OBJをクリックして選択
* 目(👁)アイコンで表示/非表示
* 選択対象の切替

---

### 7.2 Object Mode / Edit Mode の切り替え

* **Object Mode**: オブジェクト単位で選択・移動・表示切替
* **Edit Mode**: メッシュ(頂点/辺/面)を編集して削除できる

切替:

* `Tab` キーで Object Mode ⇄ Edit Mode

---

### 7.3 不要部分の削除(Edit Mode)

1. Outlinerで編集したい部屋OBJを選択
2. `Tab` → **Edit Mode**
3. 選択モードを切り替える:

   * `1`: 頂点
   * `2`: 辺
   * `3`: 面(基本はこれが楽)
4. 不要な面を選択して削除:

   * `X` → `Faces`(面削除)
   * 不要な面部分を選択できない場合は
       - `Show Overlays` を何度かクリックして反転させる
       - Edit ModeとObject Modeを行き来する
       - Object modeで削除対象のオブジェクトをクリックで選択し、Edit Modeに移動する
       - などを繰り返すと選択可能になることが多い
5. 大きく削るときは選択を補助:

   * `B`: 矩形選択(ドラッグ)
   * `C`: 円選択(ホイールで半径変更、右クリックで終了)
   * `L`: マウス下の「つながっているパーツ」を一括選択

> 実務上の流れ:
>
> * 部屋同士の重複(壁が二重になる箇所)や、スキャンのはみ出しを削る
> * “どちらの部屋で削るか” を決めてMECEにする(重複を残さない)

---

## 8. 不要部分削除で発生する「小さな破片(ゴミ)」の掃除

スキャン由来メッシュでは、削除後に「小さな破片」が残ることが多い。

### 8.1 Cleanup / Loose Geometry

* `Mesh > Clean Up > ...` 系の機能で、

  * loose geometry(孤立要素)を削除できる場合がある

ただし、破片が「面を持つ小さい島」だと、Loose判定にならず残ることがある。

---

### 8.2 最強の実務テク: 本体だけ選んで反転削除(L → Ctrl+I → X)

1. Edit Mode
2. 「残したい本体メッシュ」の上にマウスを置いて **`L`**

   * その場所と繋がっている“島”が一括選択される
3. **`Ctrl + I`**(選択反転)

   * 本体以外(ゴミ破片)が全部選択される
4. **`X` → `Faces`** で削除

> コツ:
>
> * 本体が複数の島に分かれているなら、必要な島を `Shift+L` で追加選択してから反転すると良い。

---

### 8.3 それでも取れない場合の方針

* ゴミが本体と細い糸で繋がっている場合は「島」にならないので、別アプローチが必要

  * `Merge by Distance`(微小距離で結合/整理)
  * 別途、手作業で切り離してから 8.2 を適用する

---

## 9. 鏡部分が開いて中が見える(窓化する)問題の対策

鏡の背後(誤認識部分)を削除すると、鏡位置が **開口(穴)** になり、裏の部屋が見えてしまうことがある。
対策は「穴埋め(Fill)」。

### 9.1 Edit Modeで穴の境界を作ってFill

1. 対象オブジェクトを選択 → `Tab`(Edit Mode)
2. **Edge選択モード**(`2`)
3. 穴の縁の辺を **Alt + クリック** してループ選択(閉路を作る)
4. 境界が一周選べたら **`F`(Fill)** で塞ぐ

> 注意:
>
> * `Grid Fill` は「四角っぽく整った穴」向き。スキャン穴はギザギザで失敗することがある。
> * その場合は **F(Fill)** が確実。

---

### 9.2 Outlinerが邪魔な場合: Propertiesへ切り替えて作業

穴を塞いだ面の見た目(材質)を調整したい場合、Outlinerが見当たらない/邪魔なことがある。

* 画面のエリア左上の「エディタタイプ」アイコンから

  * `Properties`(プロパティ)に切り替える
* その中の **Material Properties**(赤い球)を使って、埋めた面に専用マテリアルを割り当てる

> ポイント:
>
> * Fillした面は元テクスチャに自然に馴染まないことがあるため、
>   “塞ぎ用” の暗めマテリアルを当てるだけでも十分自然になることが多い。

---

### 9.3 Material Properties の使い方

塞いだ面だけに「塞ぎ用マテリアル」を割り当てる手順

#### 9.3.1 塞いだ面を選ぶ

* 対象オブジェクトを選択 → Tab(Edit Mode)
* 面選択(3)
* 塞いだ面をクリックして選択(複数なら Shift+クリックで追加)

#### 9.3.2 マテリアルを新規作成して割り当て

* 右側 Material Properties(今出せていますね)
* `+`(マテリアルスロット追加) を押す
* New を押す(例: MirrorBlock など名前付け)
* そのまま Assign を押す → 選択している面だけ、このマテリアルになります

#### 9.3.3 「塞いだ板」を自然に見せるおすすめ設定(つや消し)

Material > Principled BSDF で:

* Base Color: 暗めのグレー(真っ黒より少し明るい方が自然)
* Roughness: 0.9〜1.0(つや消し)
* Specular: 0.2〜0.4(控えめ)
* Metallic: 0.0

これで「穴を塞いだ板」っぽくなって、窓に見えにくいです。
最後にAssignを押して、選択した面にMaterialが反映されたことを確認する。

#### 9.3.4 鏡っぽく見せたい場合(簡易)

鏡として見せたいなら(リアル反射は重いので):

* Base Color: 暗め
* Roughness: 0.05〜0.2
* Metallic: 0.0
* Specular: 0.8前後

ただしWeb表示側では、反射は環境マップ次第なので「黒っぽい鏡」になることが多いです。
最後にAssignを押して、選択した面にMaterialが反映されたことを確認する。

## 10. .blend形式で作業を保存

Blenderの作業途中は必ず `.blend` で保存する。

* `File > Save As...`
* 例: `project_house.blend`

> コツ:
>
> * 破壊的編集(削除・統合)をする前に、バージョンを分ける
>
>   * `project_house_v01.blend`
>   * `project_house_v02.blend`

---

## 11. GLB形式でエクスポート

ブラウザで扱いやすい形式は **glTF 2.0(GLB)**。

1. `File > Export > glTF 2.0`
2. 形式は **GLB(バイナリ)**
3. 必要なら出力設定を確認(基本はデフォルトでOK)
4. 書き出し → `house.glb`

---

## 12. ブラウザで開く

### 12.1 最小構成(オフライン共有前提)

* `house.glb` をブラウザで表示するには、単純なビューワHTMLが必要
* USBで知人に渡して開いてもらう場合は、

  * `walk.html`(ビューワ)
  * `libs/`(three.js関連)
  * `house.glb`
    を同梱して、`walk.html` をダブルクリックで開いてもらう方式が実務的

### 12.2 操作例

* ファイル選択で `house.glb` を読み込む
* FPS方式ならクリックで視点固定 → WASDで歩行

---

# 付録: よくある詰まりポイント

* **Blenderで削除しても消えない**: Object Modeで削除している／面ではなく別要素を選択している／選択表示がオフ
* **黒い点々が残る**: 面は消えてるが微小破片が残存 → `L → Ctrl+I → X → Faces` が効くことが多い
* **鏡が窓になる**: 穴埋めは Grid Fill が失敗しやすい → `F(Fill)` が確実
* **Outlinerが消えた**: エリアが別エディタになっている → エディタタイプ切替で復活可能
